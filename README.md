# D-Banking Operation Dashboard & AI Sandbox

A modern, bank-grade operations dashboard and API developer sandbox simulating real-time Virtual Account and Fund Transfer operations. Engineered with adherence to the **Bank Indonesia SNAP (Standar Nasional Open API Pembayaran)** standards.

This repository serves as a comprehensive demonstration of a next-generation corporate banking interface, complete with financial analytics, an API sandbox, resilient SDK testing nodes, and AI-assisted support copilots.

## 🚀 Key Features

* **Overview Cockpit**: Real-time aggregated statistics, donut distributions, and dual-axis composed charts for monitoring transaction volumes and fees.
* **Control Panel (Ops Desk)**: Centralized monitoring of active corporate integrations, system alerts, and an interactive Support Copilot providing actionable remediation.
* **API Sandbox (Developer Portal)**: 
  * Interactive API Reference covering Virtual Account lifecycle and Fund Transfers.
  * Security guides detailing SNAP BI’s 2-step asymmetric RSA and symmetric HMAC-SHA512 auth flows.
  * Direct payload simulation, Webhook testing, and execution environments.
* **Sentinel Monitor**: Live infrastructure heartbeat monitors tracking node latencies and protocol statuses across API gateways and RTGS rails.
* **Credential Guard**: Simulation interface for public key rotatio, risk analysis, and IP threat detection.

## 🛠️ Getting Started / Guidance to Run

This project relies on **Bun** for lightning-fast dependency resolution and execution. Ensure you have [Bun installed](https://bun.sh/docs/installation) on your machine (`curl -fsSL https://bun.sh/install | bash`).

### 1. Clone & Install Dependencies
Navigate to the project directory and install the required modules:
```bash
# Install dependencies
bun install
```

### 2. Run the Development Server
Start the local Vite development server:
```bash
bun run dev
```
By default, the application will be accessible at: **http://localhost:8080** (or similar port generated by Vite).

### 3. Authentication Bypass
For rapid development and local testing, the application is equipped with a developer bypass on the authentication screen:
* **Email:** `admin@bankxyz.co.id`
* **Password:** `admin123`

### 4. Building for Production
To bundle the dashboard for production deployment:
```bash
bun run build
```
This will compile the optimized static assets into the `dist/` folder.

## 📁 Project Layout

* `src/components/layout/` — Contains global shells (DashboardLayout, Header, Sidebar). Be sure to check out the Command Palette integration!
* `src/vam/` — Core feature modules (Value Added Modules) mapping to primary navigation routes:
  * `OverviewModule.tsx`: The primary statistics gateway.
  * `OpsDeskModule.tsx`: The Control Panel and system status hub.
  * `ApiSandbox.tsx`: The interactive SNAP API documentation center.
  * `DevPortal.tsx`: System resilience and SDK nodes simulator.
  * `OnboardingModule.tsx`: Configuration and client registry.
* `src/config/navigation.ts` — The single source of truth for sidebar menu configuration.
* `src/pages/Index.tsx` — The main routing hub incorporating lazy-loaded modules.

## 🔧 Extending the Application

* **Adding New Routes**: Define your new React component, add a unique routing entry in `src/pages/Index.tsx`, and map it to a menu button inside `src/config/navigation.ts`.
* **Updating Global Colors**: Base color scales and application variables are found within `src/index.css`.
* **Modifying API Sandbox Data**: Add or mutate API endpoint data directly into exactly mapped structures within the `API_ENDPOINTS` constant stored in `src/vam/ApiSandbox.tsx`.
